# Listing Detail Pages Regeneration Guide

## Overview
The CSNExplore website uses a hybrid approach for listing detail pages:
- **Static HTML files** are generated for SEO and performance
- **Dynamic PHP routing** ensures fresh data from the database
- **URL rewriting** provides clean, SEO-friendly URLs

## How It Works

### 1. URL Structure
```
/listing-detail/{category}-{id}-{slug}.html
```

Examples:
- `/listing-detail/stays-1-hotel-renaissance-aurangabad.html`
- `/listing-detail/cars-5-luxury-sedan.html`
- `/listing-detail/attractions-2-ellora-caves.html`

### 2. Routing Flow
1. User visits: `/listing-detail/stays-1-hotel-name.html`
2. `.htaccess` rewrites to: `listing-detail.php?category=stays&id=1`
3. `listing-detail.php` fetches fresh data from database
4. Page is rendered dynamically with current information

### 3. Static HTML Files
- Located in: `listing-detail/` folder
- Generated by: `php/regenerate-complete.php`
- Purpose: SEO, fallback, and initial page load optimization

## When to Regenerate

You should regenerate listing pages when:
1. **New listings are added** to the database
2. **Listing information is updated** (name, description, images)
3. **Header/footer design changes** need to be reflected
4. **SEO meta tags** need to be updated
5. **After database migrations** or bulk updates

## Technical Details

### .htaccess Rewrite Rule
```apache
RewriteRule ^listing-detail/([a-z]+)-([0-9]+)-[a-z0-9-]+(?:\.html)?$ listing-detail.php?category=$1&id=$2 [L,QSA]
```

### Slug Generation
```php
function generateSlug($type, $id, $name) {
    $t = strtolower(trim($name));
    $t = preg_replace('/[^a-z0-9\s-]/', '', $t);
    $t = preg_replace('/[\s-]+/', '-', $t);
    $t = trim($t, '-');
    return $type . '-' . $id . '-' . $t;
}
```

## Troubleshooting

### Issue: Pages not loading
**Solution**: Check if `.htaccess` rewrite rules are enabled and `mod_rewrite` is active.

### Issue: Old content showing
**Solution**: Regenerate pages using the command above. The dynamic PHP will always show fresh data.

### Issue: 404 errors
**Solution**: Verify the listing exists in the database and is marked as `is_active = 1`.

### Issue: Images not displaying
**Solution**: Check image paths in the database. Images should be relative paths like `images/uploads/filename.jpg`.

## Best Practices

1. **Always regenerate after bulk updates** to ensure consistency
2. **Test on localhost first** before deploying to production
3. **Backup database** before major changes
4. **Check generated HTML** for any errors or missing content
5. **Monitor server logs** for any rewrite rule issues

## Production Deployment

When deploying to Hostinger:
1. Upload all files including `listing-detail/` folder
2. Ensure `.htaccess` is uploaded and active
3. Verify database credentials in `.env`
4. Test a few listing URLs to confirm routing works
5. Check that images load correctly
